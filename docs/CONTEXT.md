# CONTEXT

本项目的领域术语表。仅定义术语，不包含实现细节。

## 核心概念

- **扫描 (Scan)**：对代码仓库执行一系列规则的静态分析，不执行任何用户代码。
- **扫描器 (Scanner)**：执行扫描的引擎，由多个适配器 (Adapter) 和规则 (Rule) 组成。
- **适配器 (Adapter)**：连接外部工具或执行特定类型扫描的模块。分为内置适配器（纯 JS 实现）和外部适配器（调用 gitleaks、semgrep 等）。
- **规则 (Rule)**：单个检查项的定义，包含匹配模式、严重等级、分类和修复建议。
- **发现 (Finding)**：单条扫描结果。必须包含证据，不能只写"可能存在风险"。
- **报告 (Report)**：扫描结束后生成的输出。分两个版本：客户版（无代码细节，可交付给最终客户）和开发者版（完整 Findings，包含文件路径、行号和修复建议）。

## 风险等级

- **阻塞上线 / Critical**：一定导致审核被拒或安全事故。如生产密钥明文提交、支付回调未验签、隐私接口无声明。
- **建议整改 / High & Medium**：有风险但不一定卡审核。如依赖有已知 CVE 但非高危、敏感信息打印到日志。
- **提醒 / Low**：最佳实践缺失。如建议加 CSP 头。

## 项目类型

- **微信小程序 (WeChat Mini Program)**：第一版优先支持的扫描目标。
- **Next.js / Node.js Web Project**：第二类支持的扫描目标。
- **前端 + 后端 API 项目**：第三类支持的扫描目标。

## 报告格式

- **Markdown 报告**：v0.1 唯一输出格式。纯文本，按严重等级分组，适合开发者自用。
- **JSON 报告**：直接序列化 `ScanResult`，供脚本和工具链消费。
- **HTML 报告**：自包含静态 HTML 文件（内联 CSS/JS），双击浏览器打开，适合离线分享给非技术人员。
- **SARIF 报告**：Static Analysis Results Interchange Format (SARIF 2.1.0)，微软/OWASP 标准，供 CI/CD 系统和安全平台（GitHub Code Scanning、Azure DevOps 等）消费。

## 配置

- **配置文件 (miniapp-audit.config.json)**：放在被扫描项目根目录。控制规则开关和排除路径。不管输出格式——输出格式由 CLI `--format` 参数控制。

## 扫描历史

- **扫描历史 (Scan History)**：Web UI 中保存的最近扫描记录摘要。存储在浏览器 `localStorage`，不离开本地。每条记录包含项目路径、扫描时间、风险统计，不包含完整 Findings。

## 用户角色

- **开发者 (Developer)**：使用本工具扫描代码、查看技术详情、整改问题的人。
- **客户 (Client)**：接收最终报告的甲方或老板。只看风险汇总和是否建议上线，不看代码细节。
