# ADR 0001: CLI + Web UI 双模式交付

## 状态

已采纳

## 背景

MVP 的目标用户是独立开发者和小程序外包团队。他们既需要在本地快速扫描（贴进现有工作流），也需要在浏览器里查看和分享报告。

## 决策

第一版同时提供 CLI 工具和 Web UI，共享同一套扫描引擎。

- CLI：`npx miniapp-audit ./my-project`，输出 Markdown 报告
- Web UI：Next.js 页面，上传 zip 或指定本地路径，在线查看和导出报告

## 替代方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| 仅 CLI | 最快实现 | 无法分享报告，不适合给客户看 |
| 仅 Web UI | 报告分享方便 | 扫描前必须上传/导路径，开发体验差 |
| 双模式 | 覆盖两种场景 | 需要抽离共享引擎，架构多一层 |

## 后果

- `src/scanner/` 必须是纯逻辑，不依赖 CLI 或 Web 框架
- Web UI 和 CLI 都通过同一个 `scanProject()` 入口调用
- 报告模板需要同时支持 CLI 输出（纯文本/Markdown）和 Web 渲染（React 组件）
