# ADR 0002: 内置扫描器优先，外部工具作为可选适配器

## 状态

已采纳

## 背景

静态分析依赖外部工具（gitleaks、semgrep、npm audit CLI）能获得高准确性，但要求用户预装这些工具，增加 MVP 使用门槛。

## 决策

第一版以纯 JS 内置扫描为主，外部工具作为可选适配器。

- 内置：密钥泄露通过正则 + 文件路径匹配；依赖漏洞通过 `npm advisory API` + 本地漏洞库查询；隐私 API、支付回调通过 AST 解析
- 外部适配器：`gitleaks` 适配器、`semgrep` 适配器、`npm audit` 适配器，仅在用户已安装时可用

## 替代方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| 仅外部工具 | 准确率高、维护成本低 | 安装门槛高，用户可能放弃 |
| 仅内置 | 零依赖、开箱即用 | 第一版规则覆盖窄 |
| 内置为主 + 外部可选 | 降低门槛，保留扩展性 | 需要设计适配器接口 |

## 后果

- 适配器接口需要支持"可用性检测"，不可用时降级到内置扫描
- 扫描结果需标注来源（内置规则 vs 外部工具），方便用户判断可信度
- 内置规则的覆盖面和准确性会随着版本迭代逐步提升
