# ADR 0003: 双版本报告输出（客户版 + 开发者版）

## 状态

已采纳

## 背景

报告需要同时服务两类读者：开发者（需要文件路径、行号、技术修复建议来整改问题）和客户/甲方老板（需要通俗的风险说明和"是否建议上线"的结论）。

## 决策

扫描完成后生成两份报告：

- **开发者版**：包含所有 Findings，文件路径、行号、证据、技术修复建议、参考链接
- **客户版**：去掉代码细节，只保留风险汇总、整改清单、是否建议上线的结论、项目整体风险评级

## 替代方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| 单份分层展示 | 一份 HTML 搞定所有人 | 客户看到技术细节会困惑，开发者找不到足够信息 |
| 两份独立报告 | 各自精准匹配受众 | 相当于维护两套渲染逻辑 |

## 后果

- 报告渲染需要两套模板（客户版 Markdown/HTML、开发者版 Markdown/HTML）
- Finding 数据结构需要标记哪些字段仅在开发者版展示
- 导出时需要让用户选择导出哪个版本
